API

  1. initShaders(gl,vertexShaderId,fragmentShaderId) 这是经过封装后的api
    `
    function initShaders(gl,vertexShaderId,fragmentShaderId) {

    var vertShdr;
var fragShdr;
var vertElem = document.getElementById(vertexShaderId);
if(!vertElem) {
    alert("Unable toload vertex shader" + vertexShaderId)
    return -1
}
else {
    vertShdr = gl.createShader(gl.VERTEX_SHADER);
    gl.shaderSource(vertShdr,vertElem.text)
    gl.compileShader(vertShdr)
    if(!gl.getShaderParameter(vertShdr,gl.COMPILE_STATUS)){
        var msg = "Vertex shader failed tocompile"
        return -1
    }

}
var fragElem = document.getElementById(fragmentShaderId)
if(!fragElem) {
    return -1
} else {
    fragShdr = gl.createShader(gl.FRAGMENT_SHADER)
    gl.shaderSource(fragShdr,fragElem.text)
    gl.compileShader(fragShdr)

}
var program = gl.createProgram()
gl.attachShader(program,vertShdr)
gl.attachShader(program,fragShdr)
gl.linkProgram(program)
return program

    }
    `

  3. drawArrays(mode,first,count)
    参数： mode 指定绘制的方式，可接收以下常量符号: gl.POINTS,gl.LINES,gl.LINE_STRIP,gl.LINE_LOOP,gl.TRIANGLES,gl.TRIANGLE_STRIP,gl.TRIANGLE_FAN
    first 指定从哪个顶点开始绘制（整型数）
    count 指定绘制需要用到多少个顶点（整形数）
    返回值 无
    错误　 INVBALID_ENUM 传入的mode参数不是前述参数之一
    INVALID_VALUE 参数first或count 是负数
  4. getAttribLocation(program, name)
    获取由name参数指定的attribute变量的存储地址
    参数 program 指定包含定点着色器和片元着色器的着色器程序对象
    name 指定想要获取其存储地址的attribute 变量的名称
    返回值 大于等于0 attribute变量的存储地址
    -1 指定的attribute变量不存在，或者其命名具有gl_或者webgl_前缀
    错误 INVALID_OPERATION 程序对象未能成功连接
    INVALID_VALUE name参数的长度大于attribute变量名的最大长度(默认为256字节)
  5. vertexAttrib3f(location, v0,v1,v2)
    将数据(v0,v1,v2)传给由location参数指定的attribute变量.
    参数 location 指定将要修改的attribute变量的存储位置
    v0 指定填充attribute变量第一个分量的值
    v.. 指定填充attribute变量第..个分量的值
    返回值 无
    错误 INVALID_OPERATION 没有当前的program对象
    INVALID_VALUE location大于等于attribute变量的最大数目（默认为8）

片元着色器

片元着色器控制点的颜色。 如前述，片元就是显示在屏幕上的一个像素（严格意义上来说，片元包括这个像素的位置、颜色和其他信息）。

webGL坐标系统

以绘图区域中心为坐标原点的右手坐标系，垂直屏幕向外的是坐标系的Z轴正方向。与canvas的坐标系不一样，canvas是以绘图区域的左上角为坐标原点，y轴正方向是向下的。

Three.js学习

概念：

  1. scence(场景)
  2. camera(相机)
  3. renderer(渲染器)

场景：

Scence: 一个容器，用来保存并跟踪所有我们想渲染的物体。一般不会涉及到很复杂的操作，只需要在程序最开始的时候进行初始化即可。
方法：

  1. add在场景中添加对象
  2. children 返回场景中所有对象，包括相机和光源
  3. getChildByName 可以在创建对象的时候通过为其name属性指定一个独一无二的值，这样就能通过这个方法直接获取指定的对象
  4. remove 移除对象
  5. traverse 对所有children属性中的对象都执行一次回调函数
  6. fog设置场景中的雾化效果
  7. overrideMaterial 强制所有物体使用相同材质。

相机

camera: webgl和three.js都是使用右手坐标系的，坐标原点在canvas绘图中央。同时，照相机也需要添加进入场景之中。注意：坐标系中的长度是没有单位的。所以绝对长度不重要，只是代表相对长度。

渲染器

renderer：将内容与canvas元素进行绑定。
代码：
var renderer = new THREE.webGLRenderer( { canvas: document.getElementById(名字) } )
当定义好场景而且添加了照相机后就可以调用渲染器函数，对canvas进行一次渲染了。

几何体

生成几何体：

  1. 可以用于定义函数生成预设的几何体，
    如: 立方体（cubegeometry），平面（planeGeometry）等
    留意参数中的detail

  2. 自定义几何体

  3. 先指定几何体的顶点
    var vertis = [new THREE.vector(1,3,1)...]
  4. 再以顶点顺序标定表面
    new THREE.FACE3(0,2,1)//数字是顶点序号
  5. 生成自定义”空”几何体（无任何数据）
    geo = new THREE.Geometryc
  6. 把顶点和表面作为属性传入对象
    geo.vertices = vertices geo.faces = faces
  7. 调用对象进行几何体生成（计算重心）
    geo.compute(entroidsc)
  8. 顶点合并
    geo.mergeverticese()
  1. 高级几何体
  2. 凸面体(convexGeometry)
    给一堆顶点，生成一个包含这些顶点的凸包
  3. 管状几何(TubeGeometry)
    沿着一条三维样条曲线拉伸出来的一条管子
  4. 扫描体(LatheGeometry)
    沿着一条光滑曲线开始创建图形，就像花瓶的一条边360度扫描后生成一个花瓶。
