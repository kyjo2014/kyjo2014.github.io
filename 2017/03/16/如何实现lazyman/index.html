如何实现一个lazyman

这两天听到同学参加面试的时候被问到了这个问题，非常有趣。特别上网搜资料记录下来。
题目的要求是这样的：

实现一个LazyMan，可以按照以下方式调用:
LazyMan(“Hank”)输出:
Hi! This is Hank!
LazyMan(“Hank”).sleep(10).eat(“dinner”)
输出
Hi! This is Hank!
//等待10秒..
Wake up after 10
Eat dinner~
LazyMan(“Hank”).eat(“dinner”).eat(“supper”)
输出
Hi This is Hank!
Eat dinner~
Eat supper~
LazyMan(“Hank”).sleepFirst(5).eat(“supper”)
输出
//等待5秒
Wake up after 5
Hi This is Hank!
Eat supper
以此类推。

  1. 行为流程要进行延时操作
    一想到延时立马就想到了setTimeout函数，然而这个函数与c语言的sleep函数不一样。这个函数并不会对进程进行堵塞。而是把延时函数放在任务队列中，
    一旦执行完函数依然会立刻进行返回，从而导致接下来的函数继续立刻执行。所以要求函数的实际行为不能够与函数的执行同步。这样子应该怎么做呢？很容易，既然setTimeout是把一个函数放在任务队列中，让延时执行函数与外部函数的执行时机不一致。那么我也去实现一个任务队列就好啦。这样就可以写成这样的函数。
